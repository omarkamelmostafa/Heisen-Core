# Claude Artifacts Store

This folder stores reports, validation results, and other artifacts generated by Claude Code.

---

## How to Save Future Artifacts

### Option 1: Direct Command (Quickest)

Just tell me:

```
"Save this answer to claude/brain/artifacts/[filename].md"
```

Or:

```
"Copy this report to claude/brain/artifacts/validation-report-001.md"
```

### Option 2: Auto-Save on Validation

When running validation commands, add:

```
/speckit.validate ... and save the report to claude/brain/artifacts/[feature]-validation.md
```

### Option 3: Using Cron for Recurring Reports

If you want periodic snapshots saved:

```
Create a cron job that runs every hour and saves the git status report to claude/brain/artifacts/git-status-[timestamp].md
```

## Naming Convention

Suggested patterns:

| Pattern | Example |
|---------|---------|
| `{feature}-{type}-{date}` | `auth-loading-verification-2026-03-26.md` |
| `{phase}-{feature}` | `phase-3.5-auth-loading-verification.md` |
| `validation-{run#}-{feature}` | `validation-run5-auth-session.md` |
| `defect-analysis-{date}` | `defect-analysis-2026-03-26.md` |

## Quick Copy Commands

If you want to copy existing artifacts:

```bash
# Copy to a new name
cp phase-3.5-auth-loading-verification.md phase-3.5-backup.md

# Copy to another location
cp phase-3.5-auth-loading-verification.md ~/Desktop/
```

Or just ask me:

```
"Copy the validation report to my Desktop"
```
