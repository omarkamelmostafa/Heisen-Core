## 🗺️ Step-by-Step Refactor Plan

### **PHASE 1: CLEANUP & PREPARATION**

**Step 1.1:** Remove technical debt
- Delete `src/notFound_files/` (build artifacts)
- Delete `src/to-delete/` (legacy code)
- Audit `combine.js` files - determine if they're needed

**Step 1.2:** Update dependencies
- Add to package.json: `@reduxjs/toolkit`, `redux-persist`, `axios`, `js-cookie` or `universal-cookie`

### **PHASE 2: AXIOS HTTP CLIENT ARCHITECTURE**

**Step 2.1:** Create base axios configuration
```
src/services/api/
├── clients/
│   ├── base-client.js          # Common axios config, interceptors
│   ├── public-client.js        # No auth token
│   └── secured-client.js       # With auth token, auto-refresh logic
```

**Step 2.2:** Implement interceptor chain
- Request interceptor: add auth headers
- Response interceptor: handle success
- Error interceptor: token refresh logic

### **PHASE 3: REDUX TOOLKIT MIGRATION**

**Step 3.1:** Restructure store with RTK
```
src/store/
├── index.js                    # Store configuration + persist
├── root-reducer.js
├── root-actions.js
└── slices/                     # Replace modules/
    ├── auth.slice.js
    ├── user.slice.js
    ├── ui.slice.js
    └── api/                    # Consider RTK Query instead
```

**Step 3.2:** Configure Redux Persist
- Decide what to persist (auth, user preferences)
- Setup storage engine (localStorage vs sessionStorage)

### **PHASE 4: COOKIE MANAGEMENT LAYER**

**Step 4.1:** Create cookie service
```
src/services/storage/
├── cookie-service.js           # Wrapper around js-cookie
├── storage-constants.js        # Token names, expiry times
└── token-manager.js            # Token refresh coordination
```

### **PHASE 5: INTEGRATION & MIGRATION**

**Step 5.1:** Update existing services
- Migrate `auth-service.js`, `user-service.js` to use new HTTP clients
- Update components to use new RTK hooks and patterns

**Step 5.2:** Implement token refresh flow
- Silent refresh logic in secured-client
- Automatic retry of failed requests
- Logout on refresh failure

## 🔄 Proposed API Client Flow

```
Public Requests → public-client → API
                ↓
Secured Requests → secured-client → [add token] → API
                                  ↓
                            [401/403 response?] → pause requests → refresh endpoint
                                  ↓
                            [success?] → update token → retry original
                                  ↓  
                            [fail?] → clear tokens → dispatch logout → redirect to login
```

3. **Migration Path**: Gradual migration vs big bang? I'd recommend feature-by-feature
4. **Testing Strategy**: Mock axios adapters for testing the refresh flow
